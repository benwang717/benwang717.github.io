git版本管理工具

发表于2023-04-25|更新于2024-04-25
|阅读量:

笔记以本地仓库的操作为主, 涉及到远程仓库以后再补充. git工具的仓库管理有两种方式, 一个是通过命令行管理的git bash 另一个是更加直观的可视化工具 git gui. 我自己用后者多一点, 因为开发环境是windows, 可视化工具用起来真的很方便, 但是在别的开放系统中可能还是要用到git bash, 这里笔记就是以git bash命令行的方式对项目版本控制的常用流程做一个笔记记录.

  1. 首先新建一个文件夹, 在文件夹中创建一个txt文件, 编辑该txt文件,随便写什么都可以, 这里为了方便观察, 可在txt文件中写个0就可以.

  2. 关闭txt文件, 在文件夹中鼠标右键选择”Open Git Bash here”, 这里就开的了git bash

  3. 在弹出的bash框中, 写入 git init, 仓库就初始化完成了

  4. 键入 git add . 别忘了空格后加个. ,这里是把文件放置在就绪状态, git仓库的提交机制就是先把变更或者新增的文件放置在一个就绪的状态, 然后再提交

  5. 就绪之后, 键入 git commit -m “引号里写提交的备注信息” , 这里就是把就绪的文件提交, 这就完成了一次提交

  6. 创建子分支, 键入 git branch 分支名称 (在创建项目的时候, 会给你默认开辟一个主分支master, 在实际项目中, 有时候可客户需要一个特定的功能, 这个时候需要分支一个子分支, 不能让它影响主分支.)

    6.1 这里需要补充说明一下 “git branch 分支名称” 这种方法可以新建一个子分支 但是不会切换到你的新建的这个分支中去, 完成这个操作如果你需要切换到子分支还需要再切换到子分支, 键入 git checkout 子分支名称, 就可以切换.

    6.2 如果需要新建一个分支并且自动切换到这个分支中去需要使用 “git checkout -b 分支名称”

    6.3 如果在创建一个分支的时候不小心起错了名字或者某个子分支不再需要的时候, 使用 “git branch -d 分支名称” , 可以删除对应的分支.

    6.4 在中大型项目中, 每个人负责的可能是对应的某一个具体的功能. 这个时候就需要把某一个子分支合并到主分支中去, 首先你要切换回主分支”git chechout master”, 然后把子分支合并过来, “git merge 子分支名称” 即可.
